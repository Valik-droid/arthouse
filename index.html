We create our own website independently
Today there are three main ways: using HTML, CMS or constructor. It should be noted that the first method requires special knowledge from the user - programming languages HTML, PHP, JavaScript, CSS style sheets. Of course, information about all this can be found on the Internet and studied, but there are faster and easier ways. So let's focus on them.
The first stage
Preparatory. The point is that we just create the files, folders we need, connect the libraries and frameworks we plan to work with, create a stylesheet, a file for web scripts, and so on. We just prepare everything to get started.
The second stage
Creating a grid. This is the framework of our site. The simplest version of the grid is traditionally considered to be: a header, the main content, a block on the right or left and the footer (bottom) of the page. In your layout, everything can be more complicated. In addition, the individual blocks contain other elements that you will also need to place. But that's later, and for now you just need to create a grid. The most popular grid options can be viewed on the htmlbook. To do this, you need knowledge in the field of creating and arranging structural elements on the page. For example, you need to know what tabular and block layout is, how you can set the width and height of elements, set the flow, etc. You need knowledge that can be gained in the process of learning layout. You will be able to receive the most complete knowledge in our course on layout techniques.
The third stage
Work with smaller elements. To continue the layout of the site, you need to deal with the location of other elements on the page. The grid is just a framework, which is created by an experienced layout designer in a couple of minutes. Then begins the main part of the work - to arrange all the elements so that they were exactly as in the layout.For example, the logo should be in the upper right corner of your header. You need to know how to put it there. And in this, again, our video tutorials on the basics of HTML and CSS will help you. If you plan to make a rubber layout of the site, then you will need to consider the features of positioning elements for it. To understand the behavior of rubber blocks, I recommend taking this practical course.
The fourth stage
Can be done in conjunction with creating a grid and positioning all elements of the web page. This is a work with styles. It's great if all the elements are arranged by you correctly, but without styles they will look unpresentable. Setting the page background, width and color of icons, creating and distributing style classes and identifiers, prescribing basic properties and additional visual effects are just some of the tasks that a typesetter faces when working with styles.It may seem difficult to a beginner, but to learn all this quickly enough. HTML and CSS are not programming, you don't have to worry too much - just learn the basic rules, tags and properties once and for all and it will stay in your head for a long time. Of course, subject to constant practice.
The fifth stage
We have already made serious progress. The main part of the work is completed - ours, albeit still imperfect, but already the layout of the site. Making it even better is a further major challenge. From this stage, we begin to actively test our layout in all major browsers. The main thing for us is to see how smoothly and beautifully everything is displayed. If there are no problems, then you can be congratulated. As a rule, you can't do without mistakes, especially if you have a complex layout.Random typos in the code, the lack of cross-browser prefixes for some properties, the use of technologies that have little support - these and other factors can affect the display of the site in different browsers. Typically, most problems occur with older versions of Internet Explorer.Here it is necessary to choose: or in general to spit on display of a site in IE 6-8, or somehow to get out, applying special programs. For example, IE Tester was invented especially for this purpose. The program tests the site in older versions of this browser, showing what it would look like there. There are also special fixes that add support for some elements via JavaScript.
The sixth stage
Final inspection and bringing to perfect condition. To do this, you will need a debugger and validator, which will help identify and eliminate minor errors.
